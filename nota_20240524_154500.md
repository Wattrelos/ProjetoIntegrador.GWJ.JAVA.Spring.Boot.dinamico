# Refatoração: Implementação do Query Builder no DAO

**Implementação**: 
Criação definitiva da classe `QueryBuilder.java` contendo uma API fluente para construção de sentenças SQL (`INSERT`, `UPDATE`, `DELETE`, `SELECT` com `JOIN`). Os métodos internos de manipulação de texto do `DataAccessObject.java`, como `buildInsertSql` e `buildM2mSelectSql`, além da geração das instruções em `updateForClass` e `delete`, foram adaptados para utilizar o novo construtor de queries.

**Motivo**: 
Reduzir a complexidade ciclomática e a carga cognitiva proveniente de múltiplas concatenações de *strings* de banco de dados diretamente na classe de persistência (DAO).

**Benefício**: 
Garante que a geração do SQL e o roteamento/manipulação da API JDBC sigam caminhos independentes (Single Responsibility Principle). A classe `QueryBuilder` lida sozinha com crases (` ` `), vírgulas, espaços faltantes antes de cláusulas como `WHERE` e formatações sensíveis do banco, tornando o sistema altamente testável sem a necessidade de uma conexão com o MariaDB.
